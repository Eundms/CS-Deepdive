# Shell
- 사용자가 입력한 명령을 해석해서 커널로 전달하거나 커널의 처리 결과를 사용자에게 전달하는 역할
- GUI에서는 터미널이라는 프로그램을 이용, 외부에서 SSH로 접속하면 Shell에 접속

# 프롬프트

```shell
master@master:~$ 
# 사용자이름@호스트이름
```

- `~`는 사용자의 홈 디렉토리
- `$`이면 일반 사용자, `#`이면 슈퍼 사용자

## 명령어 입력 방식
- 대화형: 명령어를 입력하고 바로 결과를 확인
- 스크립트: 미리 파일에 명령어를 기록해두고 shell에 넘겨주는 방식

## 명령어 입력시 에러 대처
- 키보드 입력이 안 되는 경우: `CTRL + C`
- 프롬프트의 문자가 깨지는 경우: `CTRL + L`

## 명령어 구조

```shell
명령어 [옵션][인자]
```

1. 옵션: 명령의 세부기능, `-` 또는 `--`로 시작
   - `-` : 옵션이 한 글자
   - `--` : 옵션이 단어
   - 옵션은 여러 개 사용 가능, 순서 없음, 결합 가능 (`-alp`)

2. 인자: 명령에 전달하는 값, 각 요소는 공백으로 구분

## Tab 자동완성
- 입력 중인 내용으로 시작하는 명령이 1개면 tab으로 자동완성
- 여러 개인 경우 tab 두 번 누르면 모두 출력
