# history : 명령어 수행 내용 출력

- `!!` : 직전 명령을 재실행
- `!번호` : 번호에 해당하는 명령을 재실행
- `!문자열` : 문자열로 시작하는 마지막 명령을 재실행
- `history -d 라인번호` : 라인번호에 해당하는 명령어를 삭제
- `history -c` : 명령어 전체 삭제

# whereis : 명령어 위치 확인
> 설치한 프로그램의 명령어 위치 확인하기 위함

```shell

```
- b : 바이너리 파일만 검색
- m : 메뉴얼만 검색
- s : 소스 파일만 검색

# which : path에 기록된 디렉토리에서 명령어의 위치 검색

```shell

```
- `-a`, `--all` : 모든 내용을 출력

# 시스템 정보

```shell
uname [옵션]
```

- a : 시스템의 모든 정보 확인
- m : 하드웨어 정보 확인 (CPU 아키텍처)
- n : 호스트 네임
- r : 운영체제 릴리즈 번호 (커널 버전)
- s : 운영체제 이름
- v : 버전 출시 일자

```shell
hostname # 호스트 이름
arch     # 아키텍처 
env      # 환경변수
```

# 파일 내용 출력

## cat : `cat [옵션][파일]`
- 옵션 n : 라인 번호를 같이 출력
- 예시 : `cat /etc/hosts`

## more : `more [옵션][파일]`
- +행번호 형태로 시작 행 설정
- Enter → 한 행씩 이동, Spacebar → 화면 단위로 이동

## less : `less [파일]`
- j : 한 행 아래로
- k : 한 행 위로
- spacebar : 다음 화면으로
- CTRL + b : 이전 화면으로

> cat은 무한 스크롤, more/less은 페이지네이션, less는 vi편집기 방식

## tail : `tail [옵션][파일]`
- `+행번호` : 지정한 행부터 끝까지 출력
- `-숫자` : 출력 행 수 지정 (기본 10)
- `-f` : 파일 출력을 종료하지 않고 계속 출력 (로그 확인 시 유용)

# 파일 복사 : `cp [옵션][파일1][파일2]`
- `-i` : 파일2 존재 시 덮어쓸지 확인
- `-r` : 디렉토리 복사

```shell
# 일반 파일 복사
cp /etc/hosts text1

# 디렉토리로 복사
mkdir temp
cp text temp

# 새로운 이름으로 복사
cp text1 temp/text2

# 여러 파일을 디렉토리로 복사 (마지막이 디렉토리)
cp /etc/hosts /etc/services temp

# 디렉토리 복사
cp -r temp temp2
```

---

# 파일 이동과 이름 변경 : `mv [옵션][파일/디렉토리][파일/디렉토리]`
- `-i` : 이미 존재하는 경우 덮어쓸지 확인

```shell
# 이름 변경
mv text1 data1

# 파일을 다른 디렉토리로 이동
mv data1 temp/

# 여러 파일 이동 (마지막 인자는 디렉토리)
mv temp/data1 temp/data2 .

# 디렉토리 이름 변경
mv temp2 temp3
```

# 파일 링크
- 기존 파일에 새로운 이름을 붙이는 것

## 종류
- 하드 링크 : 기존 파일에 새로운 파일 이름을 추가로 생성
- 심볼릭 링크 : 원본 파일을 가리키는 새로운 파일을 만드는 것

## i-node
- 리눅스 파일 시스템은 `파일이름 + i-node + 데이터 블록`으로 구성
- i-node에는 파일 종류, 크기, 소유자, 변경 시간, 데이터 블록 주소 등 저장
- i-node 확인: `ls -i`
- i-node 번호가 같으면 같은 파일

# 디렉토리 관련 명령어

## mkdir : `mkdir [옵션] [디렉토리]`
- `-p` : 중간 디렉토리도 포함하여 생성
```shell
mkdir -p one/tmp/test
```

## rmdir : `rmdir [옵션] [디렉토리]`
- `-p` : 지정한 디렉토리 삭제 후 부모 디렉토리가 비어있으면 자동 삭제
- rmdir은 내부에 파일이나 디렉토리가 존재하면 삭제 불가
