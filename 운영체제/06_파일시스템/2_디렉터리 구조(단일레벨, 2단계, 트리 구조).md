# 디렉터리 구조

## 디렉터리란?

- 파일의 이름과 위치(inode 번호 등)를 매핑하는 **테이블**
- 파일을 조직화하고, 이름으로 파일을 찾을 수 있게 해주는 구조

## 디렉터리에서 수행하는 연산

- 파일 검색 / 생성 / 삭제
- 디렉터리 나열(ls)
- 파일 이름 변경
- 파일 시스템 순회(traversal)

## 디렉터리 구조의 종류

### 1. 단일 레벨 디렉터리 (Single-Level Directory)

```
[루트 디렉터리]
  ├── fileA
  ├── fileB
  └── fileC
```

- 모든 파일이 하나의 디렉터리에 존재
- **장점**: 구현이 간단
- **단점**: 파일 수가 많아지면 이름 충돌, 관리 어려움
- 다중 사용자 환경에서 사용 불가능

### 2. 2단계 디렉터리 (Two-Level Directory)

```
[마스터 디렉터리 (MFD)]
  ├── [사용자1 디렉터리 (UFD)] → fileA, fileB
  └── [사용자2 디렉터리 (UFD)] → fileA, fileC
```

- 사용자별로 독립된 디렉터리 제공
- **장점**: 서로 다른 사용자가 같은 파일 이름 사용 가능
- **단점**: 사용자 간 파일 공유가 어려움, 그룹화 불가

### 3. 트리 구조 디렉터리 (Tree-Structured Directory)

```
[루트 /]
  ├── home/
  │    ├── user1/
  │    │    ├── docs/
  │    │    └── fileA
  │    └── user2/
  │         └── fileB
  └── etc/
       └── config
```

- 현대 운영체제의 기본 구조
- **절대 경로(Absolute Path)**: 루트부터의 전체 경로 (`/home/user1/docs/fileA`)
- **상대 경로(Relative Path)**: 현재 디렉터리 기준 (`./docs/fileA`)
- **장점**: 효율적 검색, 계층적 그룹화
- **단점**: 파일 공유 시 복사가 필요 (링크 없이는)

### 4. 비순환 그래프 디렉터리 (Acyclic-Graph Directory)

```
[루트 /]
  ├── home/
  │    ├── user1/ → fileX
  │    └── user2/ → fileX  (같은 파일을 공유)
```

- 트리 구조 + **파일/디렉터리 공유** 허용
- 구현 방법:
  - **하드 링크(Hard Link)**: 같은 inode를 가리키는 디렉터리 엔트리 추가
  - **심볼릭 링크(Symbolic Link)**: 경로명을 저장하는 특수 파일
- **장점**: 파일 공유 가능, 저장 공간 절약
- **문제점**: 삭제 시 처리 (댕글링 포인터)
  - 하드 링크: 참조 카운트(link count)가 0이 되면 실제 삭제
  - 심볼릭 링크: 원본 삭제 시 깨진 링크(dangling link) 발생

### 5. 일반 그래프 디렉터리 (General Graph Directory)

- 비순환 그래프에서 **순환(cycle)** 까지 허용
- 순환이 발생하면 무한 루프, 삭제 시 가비지 문제
- 해결: 순환 탐지 알고리즘 필요 → 비용이 큼
- 실제 운영체제는 보통 **비순환 그래프까지만** 허용

## 하드 링크 vs 심볼릭 링크

| 구분 | 하드 링크 | 심볼릭(소프트) 링크 |
|------|-----------|---------------------|
| 가리키는 대상 | inode 직접 참조 | 경로명(문자열) |
| 원본 삭제 시 | 링크 카운트 감소, 데이터 유지 | 깨진 링크 발생 |
| 파일 시스템 경계 | 같은 파일 시스템만 가능 | 다른 파일 시스템도 가능 |
| 디렉터리 링크 | 불가 (순환 방지) | 가능 |
| 명령어 | `ln fileA linkA` | `ln -s fileA linkA` |

## 핵심 정리

- 단일레벨 → 2단계 → 트리 → 비순환 그래프 순으로 발전
- 현대 OS는 **트리 구조 + 링크(비순환 그래프)** 채택
- 하드 링크는 inode 공유, 심볼릭 링크는 경로 참조
- 순환 그래프는 가비지 문제로 실제 OS에서는 제한
