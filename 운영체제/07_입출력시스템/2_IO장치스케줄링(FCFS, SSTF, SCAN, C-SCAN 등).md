# I/O 장치 스케줄링 
## FCFS(First-Come, First-Served)
> 요청된 순서대로 처리
- 구현이 매우 간단
- 성능이 비효율적 (헤드 이동이 최적화되지 않음)

### 예제
- 요청 순서: 98, 183, 37, 122, 14, 124, 65, 67 (초기 위치: 53)
- 처리 순서: 53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67


## SSTF(Shortest Seek Time First)
> 현재 헤드 위치에 가장 가까운 요청을 우선 처리

- 평균 탐색 시간이 짧아짐
- 기아 상태(Starvation) 발생 가능 : 가까운 요청만 지속적으로 처리

### 예제
- 요청 순서: 98, 183, 37, 122, 14, 124, 65, 67 (초기 위치: 53)
- 처리 순서: 
53 → 65 (가장 가까운 요청)  
65 → 67  
67 → 37   
37 → 14  
14 → 98  
98 → 122  
122 → 124  
124 → 183  


## SCAN(Elevator Algorithm)
> 디스크 헤드가 한 방향으로 이동하며 요청을 처리, 끝에 도달하면 반대 방향으로 이동

- 기아 상태 방지 (모든 요청이 처리됨)
- 엘리베이터가 층을 오르내리며 사람을 태우는 방식과 유사

### 예제
- 요청 순서: 98, 183, 37, 122, 14, 124, 65, 67 (초기 위치: 53, 방향: 증가)
- 처리 순서: 53 → 65 → 67 → 98 → 122 → 124 → 183 → (끝) → 37 → 14


## C-SCAN (Circular SCAN)
> SCAN과 유사하지만, 끝에 도달하면 반대 방향으로 이동하지 않고 처음으로 돌아가 다시 시작

- 각 요청이 일정하게 서비스됨 (균등)
기아 상태 최소화

### 예제
- 요청 순서: 98, 183, 37, 122, 14, 124, 65, 67 (초기 위치: 53, 방향: 증가)
- 처리 순서: 53 → 65 → 67 → 98 → 122 → 124 → 183 → (끝으로 이동) → 0 → 14 → 37

## LOOK (Optimized SCAN)
> SCAN에서 끝까지 가지 않고, 마지막 요청에서 방향 변경.

- 불필요한 헤드 이동을 최소화.

### 예제
- 요청 순서: 98, 183, 37, 122, 14, 124, 65, 67 (초기 위치: 53, 방향: 증가)
- 처리 순서: 53 → 65 → 67 → 98 → 122 → 124 → 183 → (반대로) → 37 → 14

## C-LOOK (Circular LOOK)
LOOK 방식에서 끝까지 가지 않고, 다시 처음으로 돌아감
C-SCAN보다 불필요한 이동이 줄어듦

### 예제
- 요청 순서: 98, 183, 37, 122, 14, 124, 65, 67 (초기 위치: 53, 방향: 증가)
- 처리 순서: 53 → 65 → 67 → 98 → 122 → 124 → 183 → (다시 처음으로) → 14 → 37
