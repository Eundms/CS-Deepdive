# 파일 개념 및 속성

## 파일(File)이란?

- 보조 저장장치(디스크)에 저장된 관련 정보의 논리적 집합
- 운영체제는 물리적 저장장치의 특성을 추상화하여 **논리적 저장 단위(파일)** 를 정의
- 사용자 관점에서 가장 작은 저장 단위

## 파일 속성 (File Attributes / Metadata)

| 속성 | 설명 |
|------|------|
| 이름(Name) | 사람이 읽을 수 있는 유일한 정보 |
| 식별자(Identifier) | 파일 시스템 내에서 파일을 식별하는 고유 번호 (inode 번호 등) |
| 유형(Type) | 확장자 등으로 구분 (.txt, .c, .exe 등) |
| 위치(Location) | 장치 상에서 파일이 저장된 물리적 위치에 대한 포인터 |
| 크기(Size) | 현재 파일 크기 (바이트/블록 단위) |
| 보호(Protection) | 읽기, 쓰기, 실행 등의 접근 제어 정보 |
| 타임스탬프(Timestamps) | 생성·수정·최근 접근 시간 |

## 파일 연산 (File Operations)

운영체제는 파일에 대해 다음과 같은 시스템 콜을 제공한다:

1. **생성(Create)** — 저장 공간 할당 + 디렉터리에 엔트리 추가
2. **열기(Open)** — 디렉터리를 탐색하여 메타데이터를 메모리에 로드, **파일 디스크립터(fd)** 반환
3. **읽기(Read)** — 현재 파일 위치(offset)에서 데이터를 읽음
4. **쓰기(Write)** — 현재 파일 위치(offset)에 데이터를 기록
5. **재배치(Seek / lseek)** — 파일 포인터를 원하는 위치로 이동
6. **삭제(Delete)** — 디렉터리 엔트리 제거 + 저장 공간 해제
7. **잘라내기(Truncate)** — 파일 내용만 삭제하고 속성은 유지

## Open File Table

파일을 열면 운영체제는 매번 디렉터리를 탐색하지 않기 위해 **열린 파일 테이블(Open File Table)** 을 유지한다.

```
프로세스별 열린 파일 테이블  →  시스템 전체 열린 파일 테이블  →  디스크의 실제 파일
(fd, 파일 포인터, 접근 모드)     (열린 횟수, inode 정보 등)
```

- **프로세스별 테이블**: fd 번호, 현재 오프셋, 접근 권한
- **시스템 전체 테이블**: 해당 파일을 연 프로세스 수(open count), 디스크 위치 등

## 파일 유형

| 유형 | 예시 | 설명 |
|------|------|------|
| 일반 파일 | .txt, .c, .jpg | 사용자 데이터 |
| 디렉터리 파일 | / , /home | 파일 이름과 위치 매핑 정보 |
| 특수 파일 | /dev/sda, /dev/null | 장치를 파일로 추상화 (블록/문자 장치) |

## inode (Index Node)

유닉스/리눅스 파일 시스템에서 파일의 메타데이터를 저장하는 자료구조:

- 파일 이름은 포함하지 않음 (이름은 디렉터리 엔트리에 존재)
- 파일 크기, 소유자, 권한, 타임스탬프, 데이터 블록 포인터 등을 포함
- `ls -i` 명령으로 inode 번호 확인 가능

```
디렉터리 엔트리: [파일이름 → inode 번호]
inode:           [메타데이터 + 데이터 블록 포인터]
데이터 블록:      [실제 파일 내용]
```

## 핵심 정리

- 파일은 디스크 저장의 논리적 추상화 단위
- 파일 메타데이터(속성)는 inode에 저장되고, 이름은 디렉터리 엔트리에 저장
- open() → fd 반환 → read/write/seek → close() 순서로 파일 접근
- Open File Table을 통해 열린 파일의 상태를 효율적으로 관리
