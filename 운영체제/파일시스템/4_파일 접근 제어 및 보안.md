# 파일 접근 제어 및 보안

## 파일 보호의 필요성

- 다중 사용자 환경에서 파일의 **무단 접근·수정·삭제** 방지
- 접근 유형: **읽기(Read)**, **쓰기(Write)**, **실행(Execute)**

## 접근 제어 방식

### 1. 접근 제어 리스트 (ACL: Access Control List)

각 파일/디렉터리에 대해 사용자별 권한을 명시적으로 나열:

```
fileA:
  user1 → rwx
  user2 → r--
  user3 → rw-
```

- **장점**: 세밀한 권한 제어 가능
- **단점**: 사용자가 많으면 리스트가 길어짐 → 관리 비용 증가

### 2. Unix 권한 모델 (압축된 ACL)

사용자를 3개 클래스로 분류하여 간소화:

```
-rwxr-xr--  1 owner group  4096 Jan 1 file.txt
 │││ │││ │││
 │││ │││ └── Other:  r-- (읽기만)
 │││ └──── Group:  r-x (읽기+실행)
 └────── Owner:  rwx (모든 권한)
```

| 권한 | 파일에 대한 의미 | 디렉터리에 대한 의미 |
|------|-----------------|---------------------|
| r (read) | 파일 내용 읽기 | 디렉터리 목록 조회 (ls) |
| w (write) | 파일 내용 수정 | 파일 생성/삭제 |
| x (execute) | 파일 실행 | 디렉터리 진입 (cd) |

### 권한 변경 명령어

```bash
chmod 755 file.txt    # rwxr-xr-x (8진수)
chmod u+x file.txt    # owner에 실행 권한 추가
chown user1 file.txt  # 소유자 변경
chgrp group1 file.txt # 그룹 변경
```

### 3. 특수 권한 비트

| 비트 | 설명 | 예시 |
|------|------|------|
| **SetUID (4)** | 실행 시 파일 소유자 권한으로 실행 | `/usr/bin/passwd` (root 권한 필요) |
| **SetGID (2)** | 실행 시 파일 그룹 권한으로 실행 | 공유 디렉터리의 그룹 자동 상속 |
| **Sticky Bit (1)** | 디렉터리 내 파일을 소유자만 삭제 가능 | `/tmp` 디렉터리 |

```bash
chmod 4755 file    # SetUID 설정
chmod 1777 /tmp    # Sticky bit 설정
```

## 접근 제어의 구현

### 파일 시스템별 구현

| 파일 시스템 | 접근 제어 방식 |
|-------------|---------------|
| ext4 (Linux) | Unix 권한 + POSIX ACL |
| NTFS (Windows) | 상세 ACL (DACL/SACL) |
| macOS (APFS) | Unix 권한 + ACL |

### POSIX ACL (확장 ACL)

Unix 기본 권한(owner/group/other)으로 부족할 때 사용:

```bash
# user2에게 읽기+쓰기 권한 부여
setfacl -m u:user2:rw file.txt

# ACL 확인
getfacl file.txt
```

## 파일 시스템 보안 위협과 대응

### 주요 위협

| 위협 | 설명 | 대응 |
|------|------|------|
| 권한 상승 | SetUID 악용 등 | 최소 권한 원칙, 불필요한 SetUID 제거 |
| 심볼릭 링크 공격 | 링크를 통한 민감 파일 접근 | 링크 추적 제한 |
| 경쟁 조건 (TOCTOU) | 검사-사용 사이 파일 교체 | 원자적 연산 사용 |
| 디스크 직접 접근 | 파일 시스템 우회 | 디스크 암호화 |

### 파일 시스템 암호화

- **전체 디스크 암호화**: LUKS (Linux), BitLocker (Windows), FileVault (macOS)
- **파일 단위 암호화**: eCryptfs, fscrypt (ext4)
- 데이터 유출 시에도 내용 보호 가능

## umask

파일/디렉터리 생성 시 **기본 권한에서 제거할 비트**를 지정:

```bash
umask 022

# 파일 기본 권한:  666 - 022 = 644 (rw-r--r--)
# 디렉터리 기본:   777 - 022 = 755 (rwxr-xr-x)
```

## 핵심 정리

- Unix 권한 모델: Owner/Group/Other × Read/Write/Execute
- ACL은 세밀한 제어가 필요할 때 Unix 권한을 보완
- SetUID, SetGID, Sticky Bit은 특수한 접근 제어 시나리오에 사용
- umask로 새 파일의 기본 권한을 제어
- 파일 시스템 암호화로 물리적 접근에 대한 보안 확보
