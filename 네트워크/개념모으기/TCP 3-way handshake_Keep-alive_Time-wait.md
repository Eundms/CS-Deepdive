# TCP Keep-Alive (연결 유지)

> 장시간 데이터가 오가지 않더라도 연결이 살아 있는지 확인하는 기능

- TCP 수준에서 설정할 수 있고, 일정 시간(예: 2시간) 동안 통신이 없으면 keep-alive 패킷을 보냄
- 서버나 클라이언트가 죽었는지 확인할 수 있어서 좀비 커넥션 방지에 유용함


# TIME-WAIT (연결 종료 이후 대기 상태)

> 마지막 ACK 패킷이 상대방에게 유실되었을 때 재전송을 처리하고, 지연된 패킷이 새 연결에 영향을 주지 않도록 하기 위한 대기 상태

- TCP 연결 종료 시, FIN-ACK 교환 이후 클라이언트가 마지막으로 ACK를 보내면 TIME-WAIT 상태에 진입
- 일반적으로 2 * MSL (Maximum Segment Lifetime, 보통 60초~240초) 동안 유지됨
