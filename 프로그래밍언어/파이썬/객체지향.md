# OOP (객체 지향 프로그래밍)

## 1. Encapsulation : 캡슐화
데이터와 메서드를 하나로 묶고 접근제어자를 통해 접근을 제어하는 것

```python
class User:
    def __init__(self, name):
        self.__name = name # private
    @property
    def name(self):
        return self.__name
    
    @name.setter
    def name(self, value):
        if value:
            self.__name = value
```

## 2. Inheritance : 상속
- 하위 클래스가 상위 클래스의 모든 속성을 물려받는 것

## 3. Polymorphism : 다형성
- 동일한 메시지에 대하여 다르게 반응하는 성질 

1) 오버라이딩 : 상위 타입으로 참조하되 실행은 하위 타입 기준
    - Dynamic Binding (동적 바인딩)이 이를 가능하게 한다 
    - 전략패턴, DI 등 실무에서 매우 자주 사용
2) 오버로딩 : 동일한 이름의 메서드가 다양한 매개 변수로 정의됨 
    - 컴파일 시점에 결정됨 

## 4. Abstraction : 추상화
- 불필요한 세부 구현은 숨기고 핵심만 공개

# Python 메서드 종류

## 1. 인스턴스 메서드 : 객체 단위로 호출
- 해당 인스턴스 id가 self에 자동 전달됨
```python
def method(self, ...)
obj.method()
```

## 2. 클래스 메서드 : 클래스 단위로 호출
- cls는 호출한 클래스 그 자체
- 팩토리 메서드로 많이 사용됨
```python
@classmethod
def method(cls, ...)
Class.method()
```

```python
class Circle:
    pi = 3.14

    def __init__(self, radius):
        self.radius = radius

    @classmethod
    def from_diameter(cls, diameter):
        return cls(diameter / 2)

c = Circle.from_diameter(10)
print(c.radius)  # 5.0
```

## 3. 정적 메서드 : 클래스와 인스턴스 모두 호출 가능
- 클래스나 인스턴스에 의존하지 않는 유틸리티 함수
- self, cls를 받지 않음

```python
@staticmethod 
def method(...)
Class.method()
obj.method()
```

```python
class Math:
    @staticmethod
    def add(a, b):
        return a + b

print(Math.add(2, 3))  # 5
```

# bound 호출 vs unbound 호출
- Bound method : 인스턴스에 묶인 메서드 
  - obj.method() : self가 자동 전달됨
- Unbound method : 클래스에서 직접 가져온 묶이지 않은 메서드 
  - Class.method() : self를 직접 전달해야 함 
- inspect.ismethod()로 bound method인지 판단 가능

# Accessor
- 속성에 직접 접근하는 것을 권장하지 않음 
  - 대신 속성에 접근할 수 있는 함수를 따로 둬서 사용: getter, setter

## getter
- 타입이 bool인 경우 get 대신에 is를 사용하기도 한다
- get속성명 : 속성명의 첫글자는 대문자로 작성
- 로깅 작업 수행하기도 한다

## setter
- 특정한 조건을 만족하지 않는 경우 수정하지 않도록 코드를 수행하기도 한다

# 속성
> 클래스나 인스턴스 안에 데이터를 저장하기 위해 만든 변수

## 클래스 속성
- 클래스 내부에 메서드 바깥에 변수를 만들면 클래스 속성이 됨
- 클래스로 접근해서 읽기와 쓰기가 가능
- 인스턴스를 이용해서는 읽기만 가능하고, 쓰기를 하면 인스턴스 속성으로 다시 만들어짐
- 인스턴스와 클래스에 동일한 속성이 존재하면 인스턴스는 자신의 속성을 호출

## 인스턴스 속성
- 인스턴스 메서드 안에서 self.변수를 만들면 인스턴스만 접근이 가능한 인스턴스 속성이 됨
- 메서드 안에서 self 없이 만들면 지역 변수가 됨

## 특수 속성
> `__`로 시작하고 `__`로 끝나는 속성

# 생성자와 소멸자

## __init__
- 인스턴스 생성할 때 자동으로 호출되는 메서드
- 직접 작성하지 않으면 매개변수가 self 하나인 초기화 메서드가 자동으로 만들어짐
- 작성하면 제공되는 초기화 메서드는 소멸됨

## __del__
- 인스턴스가 소멸될 때 호출되는 메서드
- 매개변수는 self만 있어야 함

> 네트워크, 파일 등 외부 자원을 사용할 때 반드시 닫아줘야 한다

# Garbage Collection
- 파이썬은 내부에 GC가 존재해서 자동으로 메모리 해제를 수행
- 모든 데이터에 reference count를 부여해서 참조횟수를 저장
  - 횟수가 0이면 메모리 해제 대상
  - 데이터의 참조를 변수에 대입하면 카운트 1 증가, 가리키는 변수가 소멸되거나 다른 참조를 대입하면 1 감소
  - 참조 횟수 확인: `sys.getrefcount(객체)`

# @decorator : 메서드를 감싸서 동작을 바꾸거나 확장하는 기능

```python
def my_decorator(func):
    def wrapper():
        print("호출 전")
        func()
        print("호출 후")
    return wrapper

@my_decorator
def say_hello():
    print("안녕하세요!")

say_hello() # 호출 전 → 안녕하세요! → 호출 후
```

- 인자 있는 함수를 데코레이터로 감쌀 수 있음
```python
def debug(func):
    def wrapper(*args, **kwargs):
        print(f"함수 {func.__name__} 호출, args: {args}, kwargs: {kwargs}")
        return func(*args, **kwargs)
    return wrapper

@debug
def add(a, b):
    return a + b

print(add(3, 4))  # 출력: 7, 그리고 디버깅 정보도 같이 출력됨
```

- 데코레이터 여러개
```python
@deco1
@deco2
def func():
    print("func() 호출") # deco1(deco2(func()))
```

# __slots__
- 파이썬의 인스턴스는 클래스에 정의된 속성 외에도 필요한 경우 생성이 가능
  - 동일한 클래스로부터 만들어진 인스턴스라도 속성이 다르게 만들어질 수 있음
  - 임의 속성 추가를 방지하는 기능이 `__slots__`
- `__slots__`안에 사용가능한 속성을 저장하면 그 이외의 속성은 생성 불가

# private 멤버와 property

## private
- 속성을 생성할 때 두 개의 밑줄(`__`)을 추가하고 이름을 설정하면 Class 외부에서 접근 불가

## property
- 변수를 사용하는 것처럼 메서드를 호출하는 문법
- `변수명 = property(fget=None, fset=None, fdel=None, doc=None)`
- fget에 getter를 지정하고 fset에 setter를 지정

# 상속 (inheritance)
- 하위 클래스가 상위 클래스의 모든 것을 물려받는 것
- 상위 클래스: super 또는 base class
- 하위 클래스: sub 또는 derived class
- 단일 상속 / 다중 상속

```python
class 상위_클래스_이름:
    내용
class 클래스이름(상위_클래스_이름):
    pass
```

## method overriding
- 상위 클래스에 존재하는 메서드를 하위 클래스에서 다시 정의하는 것
- 상위 클래스의 기능이 부족해서 추가하는 것이 목적
- 하위 클래스에서 상위 클래스의 메서드 호출: `super().메서드이름()`
- 다중 상속 시: `super(클래스이름, self).메서드이름()`

## 다중 상속
- 여러 개의 클래스로부터 상속을 받는 것
- 괄호 안에 여러 개의 클래스를 나열
- 첫번째 클래스의 멤버는 `super()`로 호출
- 이후 클래스의 멤버는 `super(앞에_존재하는_클래스, self)`를 이용

- 사용하는 이유:
  - 템플릿 메서드 패턴을 구현하기 위함
  - Polymorphism 구현을 위해서 사용
