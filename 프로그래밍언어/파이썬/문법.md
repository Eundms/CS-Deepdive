# Data Type

## 1. sequential type
- 데이터를 순서대로 저장하는 자료형
- 종류
  - 컨테이너 시퀀스 : 모든 자료형의 데이터를 저장할 수 있는 시퀀스 (list, tuple, collections.deque)
  - 균일 시퀀스 : 동일한 자료형의 데이터를 저장할 수 있는 시퀀스 (str, bytes, bytearray, memoryview, array.array)
    - 동일한 자료형의 데이터를 저장하기 때문에 id 대신 실제 데이터를 저장

## 가변시퀀스와 불변시퀀스
- 가변시퀀스 : list, collections.deque
- 불변시퀀스 : tuple, str, bytes
- range를 이용해서 일정한 패턴을 갖는 숫자 시퀀스 생성 가능

## 공통된 연산
- indexing : `[]`을 이용해서 데이터의 일부분을 접근 (실제로는 `__getitem__` 메서드 호출)
  - 맨 앞은 0부터 시작, 뒤에서부터 접근할 때는 -1부터 시작
- `+` 연결, `*` 숫자만큼 반복
- `in` 데이터 존재여부 확인
- `len(데이터)` 데이터의 개수
- slicing : `[start:end:interval]` - start번째부터 end-1번째까지 interval 단위로 가져옴
  - start 생략하면 처음부터, end 생략하면 끝까지, interval 생략하면 1

## str
- 문자열 클래스 : 크기 비교 가능, 부분 수정 안 됨
- 문자열을 리턴하는 메서드가 존재하며 메서드를 연쇄적으로 호출 가능

## list
- 순서를 갖는 객체의 집합, `[]`을 이용해서 표현
- 크기는 동적으로 변경 가능
- 생성 방법: `[]`, `[데이터 나열]`, `list(__iter__가 구현된 인스턴스)`
- 데이터의 id를 기억 : 내부에서 데이터를 수정하면 연결된 데이터가 같이 수정됨
- `list.sort()` 는 내부 데이터를 정렬, `sorted()` 내장함수는 정렬 결과를 리턴

## deque
- 양방향 삽입/삭제가 가능한 양방향 큐
- FIFO 구조나 LIFO 구조를 구현할 때 효율적
- 데이터 개수를 설정해서 특정 개수를 유지하는 기능도 존재

## queue
- queue 모듈에서 동기화된 Queue, LifoQueue, PriorityQueue 제공
- 스레드 간에 안전하게 통신할 때 사용

## tuple
- 리스트와 유사하나 내부 데이터 변경 불가능
- 생성 : `(데이터 나열)`, 데이터를 나열해도 튜플로 간주
- 하나의 데이터를 가지고 튜플을 만들 때는 `데이터,` 로 표기

# 모듈과 패키지

## 모듈
- 독자적인 기능을 갖는 구성 요소, 파이썬에서는 하나의 파일

## 모듈 사용 방법
- `import 파일명` : 파일명.멤버 형태로 사용
- `from 파일명 import 속성명` : 속성만 현재 모듈로 가져옴
- `from 파일명 import *` : 모든 속성을 현재 모듈에 포함
- `import 파일명 as 새로운이름` : 새로운 이름으로 접근
- `from 파일명 import 속성명 as 새로운이름` : 속성을 새 이름으로 사용

## 패키지
- Module을 모아놓은 디렉토리
- `__init__.py` 파일을 만들어주면 됨 (내용 없어도 됨)
- 패키지에서 모듈 가져오기: `from 패키지이름 import 모듈이름`
