## 값에 의한 전달 예제 (primitive vs 참조 타입)

```java
public class PassByValue {
    static void modifyPrimitive(int x) {
        x = 10; // 호출자에 영향 없음
    }

    static void modifyArray(int[] arr) {
        arr[0] = 99; // 배열 내부 변경 -> 호출자에 반영됨
    }

    static void reassignArray(int[] arr) {
        arr = new int[] {8, 8, 8}; // 파라미터 재할당 -> 호출자의 참조에는 영향 없음
    }

    public static void main(String[] args) {
        int a = 5;
        int[] b = {1, 2, 3};

        System.out.println("Before primitive: " + a);
        modifyPrimitive(a);
        System.out.println("After primitive: " + a);

        System.out.println("Before array[0]: " + b[0]);
        modifyArray(b);
        System.out.println("After modifyArray: " + b[0]);

        reassignArray(b);
        System.out.println("After reassignArray: " + b[0]);
    }
}
```

예상 출력:
Before primitive: 5
After primitive: 5
Before array[0]: 1
After modifyArray: 99
After reassignArray: 99

요약: Java는 항상 값에 의한 전달. 원시 타입은 값 복사, 참조 타입은 참조(주소) 값이 복사된다. 참조로 객체 내부를 변경하면 호출자에 반영되지만, 파라미터 참조를 다른 객체로 재할당해도 호출자의 참조는 바뀌지 않음.
